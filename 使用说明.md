# HoshinoPanoAssistant 使用说明

HoshinoPanoAssistant（项目内部也称 MeteoAlign）用于把星野照片中的像素位置解算到天球坐标，再把照片直接重投影到新的画布。它尤其适合固定机位流星雨序列：先解算一张基准帧，批量得到每张照片的天球模型，再只导出流星所在区域，叠加到统一取景或已有全景底图中。

本文按实际任务讲解使用方法。界面上的同名按钮可直接对应文中的操作。

## 1. 先了解三个文件

软件的主要工作成果不是一个“工程文件”，而是原图旁边的几个 JSON：

- `照片名_starpairs.json`：星点配对记录。用于继续修改配对，或作为序列第一帧的基准。
- `照片名_model.json`：源图模型。记录 `源图像素 ↔ 天球方向` 的映射，是全景构图和批处理真正需要的文件。
- `照片名_Meteor.json`：流星框选。记录原图像素坐标中的一个或多个矩形区域。

另有 `mosaic_framing.json` 一类的取景 JSON，记录目标投影、方向、输出尺寸和裁剪。它用于让许多源图严格落在同一张目标画布上。

建议让原图和同名 JSON 留在同一目录。模型也保存了相对路径和绝对路径，移动整组文件后通常仍能找到原图；如果只移动 JSON、改了原图主文件名或改变了图像尺寸，关联可能失效。

## 2. 安装、启动与素材要求

### 2.1 从源码运行

在项目目录中创建环境：

```bash
conda env create -f environment.yml
conda activate hgastro
python scripts/verify_hgastro_env.py
```

启动软件：

```bash
python main.py
```

也可以使用：

```bash
python -m meteoalign
```

环境已经存在时，可用下面的命令更新：

```bash
conda env update -n hgastro -f environment.yml
```

首次启动若发现星表不完整，软件会询问是否下载。选择下载并等待完成即可；手动下载时可运行：

```bash
python scripts/download_catalogs.py
```

### 2.2 图像要求

- 星空模拟、星点匹配、序列解析和重投影支持 TIFF、JPEG、PNG。
- 流星框选额外支持常见相机 RAW；RAW 预览依赖 LibRaw。若后续还要解算星点，应先准备 TIFF、JPEG 或 PNG 版本。
- 序列解析必须读到 EXIF/XMP 原始拍摄时间，没有可用拍摄时间的文件会被跳过。程序不会用文件修改时间代替。
- 同一固定机位序列应保持机位、镜头、焦距、对焦和图像尺寸不变。单个序列按不超过一天设计，超过 24 小时会给出警告。
- 重投影输出为带透明背景的 16-bit RGBA TIFF。源图尺寸必须与 `model.json` 中记录的尺寸一致。

### 2.3 时间和地点

时间错误会直接造成星图旋转或错位。开始解算前请确认：

- `UTC 偏移` 是拍摄地时区，例如中国标准时间为 `+8`。
- 纬度北纬为正，南纬为负；经度东经为正，西经为负。
- 单图导入后，软件会尽量用 EXIF/XMP 更新拍摄时间；若原文件不带时区，则按界面中的 UTC 偏移解释。
- 序列会按拍摄时间自动排序，列表第一张就是基准帧，不一定是文件名最小的一张。

## 3. 最常用流程：固定机位流星雨序列

这一流程最终为每张素材生成模型，并将有流星的区域批量投影到同一个透明画布。

### 第一步：准备参考星空

在“星空模拟”中填写拍摄地点、UTC 偏移、传感器尺寸、焦距和镜头类型。把拍摄时间设为序列第一帧附近，调整中心方位、高度和 Roll，使模拟视野大致接近实拍画面。

这些值一开始不必非常精确，但时间、地点、镜头类型和大致朝向应正确。普通镜头选择直线镜头；鱼眼镜头选择鱼眼并填写大致视场。极限星等只影响显示和候选星数量。

“确定参考图并导出”可保存一份参考图和 JSON，方便以后恢复同一模拟视野；直接开始匹配时并不强制导出。

### 第二步：导入序列

进入“图像序列解析”，点击“导入序列图像”，一次选中同一固定机位的全部照片。

软件会：

1. 读取 EXIF/XMP 拍摄时间；
2. 跳过没有有效时间的文件；
3. 按时间排序；
4. 把第一帧载入“星点匹配”。

如果第一帧旁已有 `第一帧_starpairs.json`，软件会自动恢复它；否则需要为第一帧建立新的星点配对。

如果画面中地景、建筑、云层或灯光占比较大，建议导入序列蒙版。蒙版必须与原图尺寸完全一致：任何非黑像素都是有效区域，纯黑区域不参与自动匹配。同一个蒙版会用于整个序列，因此只适合固定构图。将蒙版命名为 `照片名_Mask.后缀` 并放在原图旁，单图导入时也可自动识别。

### 第三步：解算第一帧

进入“星点匹配”，确认真实图像是序列第一帧。

建立一对星点的方法如下：

1. 在参考星图上按住 `Ctrl`，单击一颗可确认身份的星。该星会被加入或选中于配对表。
2. 在配对表中右键该星，选择“点选位置”。
3. 在真实图像上找到同一颗星，按住 `Ctrl` 单击。软件会在点击范围内做 PSF 精修，记录亚像素星心。
4. 重复以上步骤。至少需要 4 对；实际建议先做 6～10 对，并让它们覆盖画面中心、四周和畸变明显的区域。

点选时可使用这些操作：

- 滚轮缩放预览，按住左键拖动预览。
- 在点选模式中，`Ctrl + 滚轮` 或 `Ctrl + +/-` 调整星点搜索圈大小。
- 点选模式中右键真实图像可取消本次点选。
- 右键配对表中的未匹配星，可在已有粗略模型后选择“自动配对”。
- 配对错误时，右键清除配对后重选；不再需要的参考星可以删除。

达到 4 对后，参考星图会自动配准到真实图像。打开“叠加显示”，改变透明度，检查整幅画面的星点是否一致。不要只看总 RMS：边缘是否出现同方向偏移、某一颗星是否明显离群，同样重要。

### 第四步：扩展匹配并导出基准

选择源图的基础投影模型：

- 普通摄影镜头通常选 `普通透视镜头 (TAN)`。
- 已知等距或等立体角鱼眼分别选 `ARC` 或 `ZEA`。
- 已经是全景投影的图像可选 `MER` 或 `CAR`。
- 镜头模型不明、畸变复杂，或希望主要依赖实测锚点时，可用“普适锚点插值”。此模式尤其需要让手工星点分布覆盖有效区域，避免在锚点包围范围外过度外推。

点击“自动扩展匹配”，软件会根据当前模型在预测位置附近搜索更多星点。推荐先用“软约束模式”，让自动结果改善整体拟合但不过度拉扯手工锚点；确认自动结果可靠后，也可在配对表右键改为锚点。出现误匹配时删除相应行或整组自动匹配，减小每次新增数量或搜索半径后重试。

确认结果后：

1. 点击“导出配对 JSON”，保存为 `第一帧_starpairs.json`；
2. 点击“导出映射”，生成 `第一帧_model.json`。

这两个按钮直接写到原图同目录的固定文件名，不会弹出另存为窗口。若旧文件包含的配对更多，软件会先警告。导出映射成功后，模型还会自动预载到“全景构图”。

### 第五步：批量解算后续帧

回到“图像序列解析”，点击“开始处理图像序列”。

程序把第一帧作为固定机位基准，根据各帧 EXIF 时间预测星位，并为后续帧自动匹配、拟合时间微调和生成模型。输出直接写在每张原图旁：

```text
IMG_0001.TIF
IMG_0001_starpairs.json
IMG_0001_model.json
IMG_0002.TIF
IMG_0002_starpairs.json
IMG_0002_model.json
...
```

第一帧已有 JSON 会保留；缺失时会自动补齐。第二帧及之后的同名输出会在确认后被覆盖。某一帧失败不会终止整个序列，完成对话框会列出失败文件。常见原因是云层、遮挡、星点过少、蒙版不合适或自动匹配搜索半径太小。

表中的 `δt RMS` 是序列固定机位模型的基准误差。全部帧成功后，可按需要执行“单帧结果修正”：

- “优化取景角度”：保留统一的相机 Profile，只为每帧优化姿态。适合机位有轻微晃动而镜头几何没有变化的情况，通常优先尝试。
- “单帧重新拟合”：使用每帧自己的星对重新拟合完整模型。只在个别帧确有独立形变、且星点数量与分布足够时使用。

精修会改写各帧的 `model.json`，执行前软件会确认；配对 JSON 和原始 `δt RMS` 基准仍保留。

### 第六步：标记流星

进入“框选流星”，导入要筛选的照片。此步骤可以在解算前或解算后进行，只要最终的 `_Meteor.json` 与同主文件名图像放在一起即可。

手工框选：

1. 滚轮缩放，直接按住左键拖动平移。
2. 按住 `Ctrl`，用左键拖出一个流星框；一张图可以有多个框。
3. 用前后按钮或双击列表行切换图像。
4. 点击“保存所有框选”。

框选坐标按原始图像像素保存，不受预览缩放影响。保存后生成 `照片名_Meteor.json`；若清空某张图最后一个框并再次保存，对应的 Meteor JSON 会被删除。绿色行表示当前有流星框。

自动检测需要单独的 `metdet_worker` 和 ONNX 模型。引擎未被自动找到时，在“自动检测选项”中指定 worker 可执行文件、`metdet_worker.py` 或完整 onedir 目录；模型路径留空时使用 worker 自带模型。降低置信度有助于找出暗流星，但会增加误检。多尺度层数越高、分块数越多，通常越慢。

重要：自动检测会对本次导入的所有图像重新检测，并覆盖旧检测结果；无检测框的图像会清除旧的 Meteor JSON。检测后应逐张检查误检和漏检，再手工修改并保存。

“移动流星文件”会把有框的原图及所有同主文件名关联 JSON 一起移动到目标目录，包括 `_Meteor.json`、`_starpairs.json`、`_model.json` 等。目标目录存在同名文件时，该组不会移动。执行自动检测或移动文件前，必须先保存尚未写盘的手工修改。

### 第七步：确定统一取景

进入“全景构图”，导入一张或多张 `_model.json`。多选模型适合检查覆盖范围并确定所有素材共同使用的目标画布；导入顺序不会改变各模型本身的天球位置。**需要注意的是，“计算最优分辨率”功能不支持多张**，所以需要先导入一张，大致调整得到想要的取景，计算最优分辨率再导出取景文件。

选择目标投影和视场，然后调整取景：

- 左键拖动改变画面中心；
- `Ctrl + 左键拖动` 改变 Roll；
- 滚轮或触控板缩放改变视场；
- 修改拍摄时间、地点可改变天球到地平坐标的关系，通常应保留模型记录的真实拍摄信息。

`TAN` 适合较窄的普通透视画面；`ARC`、`ZEA` 适合圆形鱼眼效果；`MER`、`CAR` 适合宽幅或 360° 全景，其中 `CAR` 常用于等距柱状全景。

可切换显示单张模型、显示全部、只显示模拟星空、覆盖范围或原图，以检查模型之间是否一致。若源图旁已有 `_Meteor.json`，勾选“仅渲染流星区域”可以预览框选效果；没有 Meteor JSON 的源图在这一页仍会按整图显示。

取景确定后：

1. 点击“计算最优分辨率”。它按源图中心角分辨率估算输出边界。
2. 设置上、下、左、右裁剪，红框表示最终范围。
3. 点击“导出取景”，保存取景 JSON。
4. 再点击“导入取景”，把刚导出的 JSON 读回来。

最后一步不是多余操作。导出重投影前，程序要求重新导入取景，以冻结并验证完整的 `ICRS → 目标像素` 变换。重新改变投影、方向、视场、分辨率或裁剪会使已导入取景失效，需要再次导出并导入。

### 第八步：批量重投影

进入“全景图批处理”，选择“天球模式”：

1. 导入上一步的取景 JSON；
2. 一次导入全部 `_model.json`；
3. 若只需要流星，勾选“只导出流星区域”；
4. 选择输出目录并开始批处理。

软件会自动在每个模型关联的原图旁查找同名 `_Meteor.json`。只导出流星区域时，没有框选的图会被跳过，框外输出为透明；不勾选时，每张源图都完整重投影。

输出名类似：

```text
IMG_0001_model_mosaic_12000x6000.tif
```

已有同名文件不会被覆盖，软件会自动在文件名后加序号。

## 4. 单张照片的简化流程

只处理一张星野或一张全景底图时，不需要先导入序列：

1. 设置时间、地点和大致相机视野。
2. 在“星点匹配”中导入单张图像。若同目录已有 `_starpairs.json` 或 `_Mask.*`，软件会自动尝试载入。
3. 手工匹配分布均匀的星点，检查叠加，再自动扩展匹配。
4. 导出配对 JSON 和映射，得到 `_model.json`。
5. 在“全景构图”中确定目标投影，计算分辨率，导出并重新导入取景。
6. 点击“导出重投影图”。

“全景构图”可以同时导入多张模型用于共同预览和设计取景，但单次“导出重投影图”只处理当前的单张源模型。要用同一取景输出多张文件，请使用“全景图批处理”。

若只想输出流星框，先保存 `_Meteor.json`，再勾选“仅渲染流星区域”。在单图构图页中，如果没有流星 JSON，该选项会退回到整图导出；在批处理页的“只导出流星区域”中，没有框的图则会直接跳过。

## 5. 借用已有模型，减少手工找星

### 5.1 复用 Camera Profile

如果已有同一相机、镜头、焦距、对焦状态和输出尺寸的 `_model.json`，可在“星点匹配”中点击“导入 Profile”，直接从模型提取镜头内参与畸变：

- “Profile 求解 Pose”只求当前照片的相机朝向，适合稳定、可复用的镜头标定。
- “Profile 求解 Pose + 局部残差”在 Profile 上再拟合局部修正，适合镜头几何大体一致但仍有小范围残差的情况。
- “不使用 Profile”完全按当前星对重新求解。

Profile 不包含拍摄姿态、时间或地点。跨相机、跨焦距、重新对焦、裁切或改变分辨率时不能复用。

### 5.2 用相邻已解算图像做粗略取景

当当前图像与一张已解算照片有重叠，但很难直接从模拟星图认星时：

1. 导入当前未解算图像 B；
2. 在“相邻图像粗略取景”中导入相邻已解算图像 A 的 `_model.json`；
3. 星空重叠明显或星空占比面积大时选“星点对齐模式”，地景纹理更可靠时选“地景对齐模式”；
4. 点击“计算粗略取景”，如果成功计算(RMS在十几个像素以下)，参考星图即会变到图像B的星空区域，并可勾选“同步”与“叠加显示”；
5. 参考星图大致叠放后，再手工确认至少 4 颗真实星并完成正式求解。

粗略取景只是初始化，不代替最终星点验证。两张图应有足够重叠；星点模式要求可靠的共同星点，地景模式要求清晰且没有明显移动的地面纹理。参数异常时可通过旁边的“设定”调整当前模式参数。

## 6. 对齐到已有全景底图

如果已经有一张 PTGui 等软件生成的全景底图，希望把流星照片投到它的像素坐标中，可使用批处理的“底图模式”：

1. 先把底图当成一张普通源图，在“星点匹配”中解算并导出底图的 `_model.json`。如果底图本身是等距柱状或墨卡托投影，选择相应基础投影；若存在拼接局部形变，可增加分布均匀的锚点或使用普适锚点插值。
2. 分别解算要贴入的流星照片，得到它们的 `_model.json`。
3. 在“全景图批处理”选择“底图模式”。模式一旦已有导入便会锁定；需要切换时先点“清除导入”。
4. “导入底图 JSON”选择底图的 `_model.json`。
5. “导入图像 JSON”多选流星照片的 `_model.json`。
6. 按需勾选“只导出流星区域”，开始批处理。

底图模式使用底图模型的原始宽高作为目标画布，并直接采用底图的 `天球方向 ↔ 像素` 映射，不需要另做取景 JSON。输出仍是每张素材各自一张透明 TIFF，便于在图像软件中以相同像素尺寸叠加到底图。

## 7. 质量设置怎么选

### 7.1 预览贴图网格

“全景构图”中的贴图网格控制交互预览的几何精度。长边点数越多，弯曲边缘越准确，但预览越慢。它主要影响预览，不等同于最终导出的“投影网格大小”。

### 7.2 导出投影网格大小

导出页和批处理页的“投影网格大小”单位为源图像素。数值越小，映射越精细、速度越慢；默认值通常适合作为首次输出。若检查到弯曲边缘、亮星或细流星附近有重映射误差，再逐步减小。

### 7.3 修复亮星小黑点

“修复亮星小黑点（较慢）”会对缺失或低权重像素做额外精确反算。鱼眼、强畸变或很小的投影网格仍出现亮星周围黑点时再开启；它会明显增加导出时间。

预览卡顿时，可降低 `preference.json` 中的 `mosaic_texture_scale_percent` 或 `mosaic_texture_max_long_side_px`，并降低预览贴图网格点数。修改偏好后重启软件最稳妥。

## 8. 常见问题

### 四颗星后可以拟合，但边缘错得很远

四颗只是最低要求。增加分布到画面边缘的手工锚点，确认基础投影类型正确，再扩展自动匹配。不要把星点都选在中心或一条直线上。对于复杂畸变，使用普适锚点插值或已知投影加局部残差。此外，注意避免任何亮星聚集区例如昴星团的配对(手工配对避免，自动扩展匹配选中了就删掉)。

### 自动扩展匹配加入了错误星点

删除错误行或该次自动匹配组，缩小搜索半径和每次新增数量后重试。先改善手工基准和蒙版，再依赖自动匹配。软约束比直接加入硬锚点更适合不确定的自动结果。

### 序列中部分图片被跳过

导入阶段跳过通常是没有有效 EXIF/XMP 原始拍摄时间；处理阶段失败通常是云层、遮挡、星点太少或预测搜索范围不足。程序会继续处理其他帧，可按完成提示逐张检查失败文件。

### 序列处理时RMS有突变

这一般说明序列中的取景角度有变化。单个序列需要保持取景完全不变(不使用赤道仪，相机朝向在地平坐标系下不变)，RMS异常增大表示取景角度变了，可以尝试使用“单帧取景修正”，如果RMS还是拉不回来，就把RMS突变后的帧单独作为一个新的序列。

### 导入模型后找不到原图

把 `_model.json` 与同主文件名原图放回同一目录。软件会优先寻找同目录的 TIFF、PNG、JPEG，并兼容 JSON 中保存的相对路径和旧绝对路径。确保图像主文件名和尺寸未改变。

### 为什么导出取景后仍不能导出 TIFF

必须把刚导出的取景 JSON 再导入一次。任何影响目标几何的改动都会清除这份已验证状态，改完后需重新导出、重新导入。此外，“全景构图”页面里的“导出重投影图”并不支持多张导出，多张导出请用全景图批处理功能。

### 只导出流星区域后，画面仍出现整张原图

在“全景构图”单图导出中，没有有效 `_Meteor.json` 时会退回整图。确认 JSON 与模型所关联原图同主文件名、同目录，且包含至少一个框。若需要严格跳过无框图，使用批处理页的“只导出流星区域”。

### 自动流星检测按钮不可用

等待状态栏显示检测引擎就绪；若仍不可用，在检测选项中指定 `metdet_worker` 和必要的 ONNX 模型。若当前有未保存的手工框选，也必须先点击“保存所有框选”。

### 星表下载失败

检查网络后重启并重试，或运行 `python scripts/download_catalogs.py`。星表必须位于项目的 `catalog/` 目录；详细来源见 `docs/catalog_downloads.md`。

## 9. 配置与备份建议

`preference.json` 位于源码项目根目录；打包版本中位于程序旁。它支持 `//` 和 `/* ... */` 注释，保存界面字号、默认地点、自动匹配、相邻图像配准、流星检测和全景输出等参数。软件会自动补入新版本缺少的配置项，不覆盖已有值。

长期项目建议备份：

- 原始或用于解算的 TIFF/PNG/JPEG；
- 每张图的 `_starpairs.json` 和 `_model.json`；
- 每张流星图的 `_Meteor.json`；
- 最终采用的取景 JSON；
- 如有定制，备份 `preference.json`。

有了这些文件，即使以后重新设计目标投影，也不必重新做全部星点匹配。
