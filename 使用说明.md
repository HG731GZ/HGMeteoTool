# HoshinoPanoAssistant（星野全景助手）使用说明

## 1. 软件简介

HoshinoPanoAssistant是一款面向**星野摄影**（尤其是流星雨摄影）的离线桌面软件。它的核心任务是：**告诉你照片里的每一颗星对应天上的哪个位置，并让你以任意投影方式重新组合这些图像**。

### 典型应用场景

- 拍摄了几百张流星素材，需要把流星精确贴到一张全景底图上
- 拍摄了一些全景照片的单张图，想精确地贴在天球上
- 想用不同的投影（墨卡托、立体投影、等距柱状……）展示同一组星空素材

### 工作流程概览

```
星空模拟 → 导入真实图像 → 星点匹配求解 → 导出源图模型 → 自由投影拼图 / 批处理到底图
```

软件共分五个功能标签页，分别对应上述流程的各个阶段。

## 2. 安装与运行

### 2.1 环境配置

推荐使用项目自带的 `environment.yml` 创建 conda 环境：

```bash
conda env create -f environment.yml
conda activate hgastro
```

如果环境已存在，可以更新：

```bash
conda env update -n hgastro -f environment.yml
conda activate hgastro
```

可选：验证环境是否正确：

```bash
python scripts/verify_hgastro_env.py
```

### 2.2 启动程序

在项目根目录运行：

```bash
python main.py
```

或使用包启动方式：

```bash
python -m meteoalign
```

### 2.3 星表数据

程序依赖 `catalog/` 目录下的离线星表数据（含 Yale BSC、Hipparcos、IAU CSN 及中文星名表）。首次启动时如检测到星表文件不完整，会自动弹出下载对话框引导下载。请保持 `catalog` 文件夹与源码一起放在项目根目录。

### 2.4 偏好设置

程序根目录下的 `preference.json` 文件包含所有可配置参数，支持 JSONC 风格注释。

## 3. 界面总览

软件主窗口包含六个标签页，每个标签页左侧为控制面板，右侧为预览区域：

| 标签页 | 功能 |
|--------|------|
| **星空模拟** | 根据时间、地点、相机参数生成模拟星图 |
| **图像序列解析** | 批量导入和处理图像序列 |
| **星点匹配** | 交互式星点配对、求解源图映射模型 |
| **框选流星** | 标记单张图中的流星区域，供后续按区域计算与导出 |
| **全景构图** | 导入已求解模型，自由投影拼图 |
| **全景图批处理** | 批量将多张图像贴到全景底图上 |

## 4. 星空模拟（标签页一）

此页面用于根据观测参数生成模拟星图，可作为后续星点匹配的参考。

### 4.1 观测信息

| 参数 | 说明 |
|------|------|
| **拍摄时间** | 观测的本地日期和时间 |
| **UTC 偏移** | 本地时区相对 UTC 的小时偏移（如北京时间填 +8） |
| **纬度** | 观测地点纬度，北纬为正（-90° ~ +90°） |
| **经度** | 观测地点经度，东经为正（-180° ~ +180°） |
| **海拔** | 观测地点海拔高度（米） |

### 4.2 相机参数

| 参数 | 说明 |
|------|------|
| **传感器宽 / 高** | 相机传感器的物理尺寸（毫米），如全画幅为 36×24 mm |
| **切换横/竖幅构图** | 点击交换传感器宽高值 |
| **导出图像宽 / 高** | 模拟星图导出时的像素分辨率 |
| **焦距** | 镜头焦距（毫米） |
| **镜头类型** | 直线镜头（普通透视）或鱼眼镜头 |
| **鱼眼视场** | 选择鱼眼镜头时生效，设定鱼眼镜头的对角视场角 |
| **极限星等** | 模拟星空中显示的最暗星等，数值越大星越多 |

### 4.3 视野方向

| 参数 | 说明 |
|------|------|
| **中心方位角** | 镜头指向的方位角，0°=北，90°=东，180°=南，270°=西 |
| **中心高度** | 镜头指向的高度角，0°=地平线，90°=天顶 |
| **Roll** | 相机绕光轴旋转角（°） |

### 4.4 参考图

| 控件 | 说明 |
|------|------|
| **标注模式** | "固定标注星数"：始终标注指定数量的最亮星；"固定星等上限"：标注亮于指定星等的所有星 |
| **标注星数** | 固定标注星数模式下，模拟图中标注的亮星数量（3~40） |
| **标注星等上限** | 固定星等上限模式下，只标注亮于此星等的星 |
| **确定参考图并导出** | 生成模拟星图并导出为图像文件，同时生成包含星点坐标的 JSON 文件 |
| **导入预览 JSON** | 导入之前导出的参考图 JSON，恢复星图显示 |

此页面用于批量导入和处理一组星空照片。

### 5.1 基本操作

|------|------|

| **开始处理图像序列** | 对序列中的所有图像执行星点检测等处理（需先导入序列） |

## 6. 星点匹配（标签页三）

这是软件的核心功能页面。在此页面中，你将真实拍摄的星空图像与模拟参考星图进行星点配对，从而求解出图像像素与天球坐标之间的映射关系（源图模型）。在导入序列后，序列中最早拍摄的那张图会出现在这个页面中。

### 6.1 真实图像

| 按钮 | 说明 |
|------|------|
| **导入单张图像** | 导入一张真实拍摄的星空照片 |
| **导入 Profile** | 导入相机标定 Profile 文件（JSON 格式，包含内参和畸变参数） |

导入图像后，左侧面板会显示文件路径、原始尺寸、序列状态和 Profile 信息。

**Profile 复用方式**（选择如何使用标定 Profile）：

| 选项 | 说明 |
|------|------|
| 不使用 Profile | 不使用标定数据，完全依赖星点匹配求解 |
| Profile 求解 Pose | 使用 Profile 中的内参/畸变信息，仅求解相机指向 |
| Profile 求解 Pose + 局部残差 | 在 Profile 基础上额外用薄板样条修正局部残差 |

### 6.2 星点匹配工作区

页面右侧分为上下两个预览区：

- **上方：参考星图** — 显示根据观测参数生成的模拟星空，带有亮星标注
- **下方：真实图像** — 显示导入的真实照片

两个视图均支持鼠标拖拽平移和滚轮缩放。

#### 预览区工具栏

参考星图工具栏：

| 控件 | 说明 |
|------|------|
| **同步** | 勾选后，参考星图和真实图像视图联动平移/缩放 |
| **标注** | 切换参考星图上的星名标注显示/隐藏 |
| **叠加显示** | 将参考星图以半透明方式叠加到真实图像上 |
| **透明度** | 调节参考星图叠加的透明度（0%~100%） |

真实图像工具栏：

| 控件 | 说明 |
|------|------|
| **蒙版** | 切换天空蒙版的显示/隐藏 |
| **标注** | 切换真实图像上的星点标注显示/隐藏 |

### 6.3 交互式星点配对

这是求解源图映射的核心操作：

1. **在参考列表中**：点击选中一行，右键，"点选位置"
2. **切换到真实图像**：在真实照片中找到同一颗星的位置，按Ctrl+左键选中
3. 程序会自动在点击位置附近执行 PSF（点扩散函数）精修，将星点中心定位到亚像素精度
4. 一对星点匹配完成，显示在左侧的配对表中

> **提示1**：选星时使用鼠标滚轮可调整选星圆圈的直径，触控板双指捏合同样支持。圆圈越大，PSF 搜索范围越大。
> **提示2**：想要匹配列表中没有的星，只需在参考星图中Ctrl+左键，该星即加入列表中的手动匹配组中。

### 6.4 参考星配对表

表格显示所有已建立的星点配对，包含：

| 列 | 说明 |
|------|------|
| 序号 | 配对编号 |
| 星名 | 参考星的名称（如"天狼星"、"Vega"等） |
| 模式 | 手动匹配或自动匹配 |
| 残差 | 该匹配点在天球模型中的残差（像素） |

表格支持右键菜单：折叠/展开手动/自动匹配分组、删除自动匹配组、更改匹配模式。

配对表下方按钮：

| 按钮 | 说明 |
|------|------|
| **导出配对 JSON** | 将当前配对会话保存为 JSON 文件，可后续导入恢复 |
| **导入配对 JSON** | 从 JSON 文件恢复之前的配对会话 |
| **清除所有匹配** | 清空所有已建立的星点配对 |

### 6.5 源图映射求解

建立至少 4 对匹配后，可进行映射求解。

#### 蒙版

| 按钮 | 说明 |
|------|------|
| **导入蒙版** | 导入天空蒙版（黑白图像，白色=天空区域），蒙版外的区域在自动匹配时会被忽略 |
| **清除蒙版** | 移除已导入的蒙版 |

#### 映射模型与自动匹配

| 参数 | 说明 |
|------|------|
| **基础投影模型** | 选择源图的天球映射模型。选项包括：普适锚点插值（推荐，适用于任意镜头）、普通透视镜头(TAN)、等距鱼眼(ARC)、等立体角鱼眼(ZEA)、墨卡托(MER)、等距圆柱(CAR) |
| **每次新增** | 每次自动扩展匹配时新增的候选星数量 |
| **新增约束** | 锚点模式：新增星作为硬锚点加入模型；软约束模式：新增星以较低权重参与求解 |
| **软约束权重** | 软约束模式下新增匹配点的权重（0.01~1.00） |
| **搜索半径** | 在预测位置周围搜索匹配星点的半径范围（像素） |

| 按钮 | 说明 |
|------|------|
| **自动扩展匹配** | 基于已有匹配求解的模型，自动在真实图像中搜索更多星点并建立配对 |
| **导出映射** | 将最终求解的源图模型导出为 JSON 文件，供后续全景构图使用 |

#### 自动配准辅助

底部状态栏显示当前映射模型的拟合残差信息。至少配对 4 颗星后，程序会自动计算并更新配准变换。

> **推荐工作流程**：手动配对 4~6 颗亮星 → 选择合适的投影模型 → 自动扩展匹配获取更多星点 → 检查残差分布 → 导出映射模型。
> **注意**：如果图中包含地景等干扰元素，请使用蒙版排除，否则自动扩展匹配可能出问题。蒙版必须是二值化的，黑色表示不参与计算。

### 6.6 框选流星

此页面位于“星点匹配”右侧，用于在单张图像中圈出包含流星的区域。后续导出可以只处理这些原图像素区域，减少不必要的像素到 ICRS 与渲染计算。

1. 点击 **导入流星图片**，可一次选择一张或多张 TIFF、JPEG 或 PNG 图像。
2. 在右侧预览中，使用滚轮缩放、直接按住左键拖动平移。按住 Ctrl 后以左键拖动可创建蓝色流星框；可重复创建多个框。左侧列表中流星数非零的图像行会以绿色标记。
3. 使用 **◀ / ▶** 或双击左侧图片列表中的行切换图像；列表的“流星数”是该图当前框选数。
4. **清除框选** 位于预览区右下角，只清除当前图像的所有框。左侧列表下的 **保存所有框选** 会一次保存所有“流星数”非零的图像；每个 JSON 位于相应图像同目录，例如 `IMG_1234.TIF` 对应 `IMG_1234_Meteor.json`。

JSON 中的 `top_left` 与 `bottom_right` 坐标始终是取整后的原始图像像素坐标，而不是屏幕或预览缩略图坐标。

## 7. 全景构图（标签页五）

导入已求解的源图模型后，在此页面选择目标投影方式、调整取景范围、预览和导出重投影图像。

### 7.1 源图模型

| 按钮 | 说明 |
|------|------|
| **导入单张模型** | 导入一个源图模型 JSON 及对应的原始图像 |
| **导入多张模型** | 批量导入多个模型 JSON（程序自动查找对应的图像文件） |

导入后显示模型路径、关联图像和模型状态信息。

### 7.2 拍摄信息

设置此次拼接的观测参数（时间、UTC 偏移、经纬度、海拔）。这些参数用于正确计算天球坐标。

| 按钮 | 说明 |
|------|------|
| **回到 JSON 设置** | 将拍摄信息重置为模型 JSON 中记录的值 |

### 7.3 目标投影

| 参数 | 说明 |
|------|------|
| **模式** | 目标输出投影类型：普通透视(TAN)、等距鱼眼(ARC)、等立体角鱼眼(ZEA)、墨卡托(MER)、等距圆柱(CAR) |
| **视场** | 输出图像的视场角（5°~360°），全景图通常设为 360° |
| **中心方位** | 输出图像中心指向的方位角 |
| **中心高度** | 输出图像中心指向的高度角 |
| **Roll** | 输出图像的旋转角 |
| **极限星等** | 模拟星空显示的极限星等（用于参考网格叠加） |
| **贴图网格** | 源图贴到全景图时的网格精度，单位为"点数/长边"。数值越大贴图越精细，但渲染越慢。修改后需点击"重新求解" |
| **网格** | 勾选后在预览中显示坐标网格 |

| 按钮 | 说明 |
|------|------|
| **重新求解** | 按当前网格精度重新计算贴图映射 |
| **回到源图中心** | 将视图重置回源图的原始指向 |

### 7.4 预览区

右侧预览区实时显示当前投影下的拼图效果。工具栏选项：

| 控件 | 说明 |
|------|------|
| **选择显示单张图像** | 下拉选择单独显示某张源图，或"显示全部" |
| **只显示模拟星空** | 勾选后隐藏源图贴图，仅显示模拟星空背景（方便检查星点对齐） |
| **显示范围 / 显示原图** | 切换预览叠加模式：显示各源图的覆盖范围框，或显示源图内容 |
| **透明度** | 源图叠加显示时的透明度 |

预览区支持鼠标拖拽**平移视图**（在非固定视场的投影模式下）和滚轮缩放。

### 7.5 输出取景

| 按钮 | 说明 |
|------|------|
| **计算最优分辨率** | 根据源图原始分辨率自动计算输出图像的最佳像素尺寸 |
| **导出取景** | 将当前取景参数（投影类型、视角、裁剪等）保存为 JSON |
| **导入取景** | 加载之前保存的取景参数 |

#### 裁剪设置

设置输出图像四边的裁剪像素数（上、下、左、右）。可用于裁掉边缘的未覆盖区域。

#### 导出选项

| 参数 | 说明 |
|------|------|
| **投影网格大小** | 导出时源图映射网格的像素大小。数值越小越精细，但计算越慢（默认 16 px） |
| **修复亮星小黑点（较慢）** | 勾选后额外精确反算缺失和低权重像素，用于修复鱼眼镜头亮星周围的小黑点。会显著增加导出时间 |

| 按钮 | 说明 |
|------|------|
| **导出重投影图** | 按当前所有设置生成最终的重投影图像（支持 16-bit TIFF 格式） |

## 8. 全景图批处理（标签页六）

此页面用于批量将多张已求解的源图图像贴到同一张全景底图上。

### 8.1 批处理设置

| 参数 | 说明 |
|------|------|
| **处理模式** | 天球模式：基于统一的天球坐标映射；底图模式：基于已有全景底图进行对齐 |

| 按钮 | 说明 |
|------|------|
| **导入取景 JSON** | 导入定义输出全景范围的取景 JSON 文件 |
| **导入图像 JSON** | 导入包含多张图像模型路径的 JSON 文件 |
| **清除导入** | 清除已导入的取景和图像列表 |

### 8.2 单张图像列表

导入图像模型 JSON 时，程序会自动查找其原图同目录的 `图片名_Meteor.json`。表格显示文件名、流星框选状态（无 / 有及框数 / 异常）和处理状态。

| 控件 | 说明 |
|------|------|
| **投影网格大小** | 源图按固定网格投到输出图；数值越小越精细但越慢 |
| **修复亮星小黑点（较慢）** | 对缺失和低权重像素执行额外精确反算，改善鱼眼亮星附近的小黑点 |
| **只导出流星区域** | 只处理有流星框选的图像，并将框外输出设为透明；没有框选的图像会跳过 |

| 按钮 | 说明 |
|------|------|
| **开始批处理** | 按当前网格、修复和流星区域选项批量处理图像 |

## 9. 快捷键与操作技巧

### 9.1 通用视图操作

| 操作 | 效果 |
|------|------|
| 鼠标滚轮 | 缩放视图（可在 preference.json 中禁用） |

### 9.2 星点匹配操作

| 操作 | 效果 |
|------|------|

## 10. 文件格式说明

### 10.1 源图映射模型 JSON

由"星点匹配"页面导出，包含：
- 源图像素 ↔ 天球方向的完整双向映射
- 观测参数（时间、地点）
- 相机参数（焦距、传感器尺寸、镜头类型）
- 所有星点配对记录
- 映射残差统计

### 10.2 星点配对会话 JSON

由"星点匹配"页面导出/导入，包含：
- 所有手动和自动星点配对
- 参考星信息（名称、坐标、星等）
- 图像空间中的星点位置
- 配对分组信息

### 10.3 取景 JSON

由"全景构图"页面导出/导入，包含：
- 目标投影类型和参数
- 视场角和指向
- 输出分辨率和裁剪设置

### 10.4 相机标定 Profile JSON

包含相机的内参矩阵、畸变系数等标定数据，可在星点匹配时导入以辅助求解。

## 11. 常见问题

### Q: 启动时提示星表文件不完整？
A: 程序会自动弹出下载对话框。确保网络连接正常，点击下载即可。也可手动将星表文件放入 `catalog/` 目录。

### Q: 星点匹配不准确怎么办？
A: 
1. 确保手动配对的 4~6 颗星分布均匀，覆盖图像的不同区域
2. 选择合适的投影模型——普通镜头选"普通透视镜头(TAN)"，鱼眼镜头选对应的鱼眼模型，不确定时选"普适锚点插值"
3. 适当增大搜索半径
4. 使用自动扩展匹配增加匹配点数量
5. 检查并删除残差异常大的配对点（右键配对表操作）

### Q: 全景导出图像中有小黑点？
A: 这是鱼眼镜头重投影时的常见现象。导出前勾选"修复亮星小黑点（较慢）"选项可以修复，但会显著增加导出时间。

### Q: 全景构图预览很卡？
A: 
1. 降低 `mosaic_texture_scale_percent` 参数（如改为 10~15%）
2. 降低贴图网格精度（减少长边点数）
3. 减少同时显示的源图数量

### Q: 如何批量解算多张流星照片的坐标？
A: 
1. 先在"图像序列解析"中导入同一取景的照片
2. 在"星点匹配"中给序列第一张图做匹配(导入序列后该页面会自动出现第一张图)
3. 在"星点匹配"中导出配对JSON与映射JSON
4. 回到"图像序列解析"，点击"开始处理图像序列"

### Q: 如何把单张流星/全景图的单张照片贴到固定取景上？
A:
1. 首先，确保你有所有图的model.json
2. 在"全景构图"页面导入单张model.json
3. 选择你想要的投影方法、视野范围等，拖拽也能改变取景视角，如果发现地平线不准可调整拍摄时间、地点
4. 点击"计算最优分辨率"，并调整四个裁剪选项得到你想要的取景范围
5. 点击"导出取景"得到取景JSON，再"导入取景"把这个JSON文件重新导入一下
6. 点击"导出重投影图"即可把单张图贴到全景图视野中

### Q: 如何批量把单张流星/全景图的单张照片贴到固定取景上？
A:
1. 首先，确保你有所有图的model.json与一个取景JSON
2. 在"全景图批处理"页面选择"天球模式"并导入取景JSON
3. 导入所有单张图像的model.json
4. 点击"开始批处理"

### 支持的投影类型

| 缩写 | 全称 | 适用场景 |

| MER | 墨卡托（Mercator） | 全景星图、星图软件常用 |
| CAR | 等距圆柱（Equirectangular / Plate Carrée） | 360° 全景图、VR 内容 |
